# Data releases

Every export run appends an entry here (generated by `scripts/export.mjs`). Full line-by-line history is in git.

## 2026-07-03 — schema v2.1: entry-level evidence & split URLs

- **`curated-labels.json` evidence standardized per entry**: each entry now carries its own `evidence_type` (+ `cluster_method`). The 7 Stake BTC entries are now honestly labelled `confirmed_deposit` (1) / `behavioral_cluster` via `sweep_flow` or `common_input_ownership` (6) instead of inheriting the "name-tag" class — the file's own `_about` previously overclaimed this and has been corrected.
- **Seed/derived recount** from the fix: 1,228 seed + 2,040 derived (the 6 behavioural Stake BTC entries moved to `derived_wallets`, `confidence_score` 92 → 70; the confirmed deposit address stays seed at 90).
- **`evidence_url` split** into `address_explorer_url` (the address page — balances/txs, always present) and `evidence_source_url` (where the attribution evidence lives: explorer page for name-tags, our public Dune query `dune.com/queries/7810326` for label-set imports, `intel.arkm.com` entity view for Arkham, published methodology for behavioural attributions). Curated entries additionally carry `source_note` with the human-readable evidence incl. anchoring transactions.
- **Rule parameters published** in DATA_DICTIONARY.md → "Published rule parameters": volume-suspect thresholds (scrutiny floor $50M/7d, avg-transfer ceiling $50K/tx, per-counterparty ceiling $50K, manual flag list), verified-volume rules (cp_internal exclusion, attributed-only, suspect-excluded, mainstream tokens), and wallet-role inference thresholds. Every flag is now recomputable from public data.

## 2026-07-03 — schema v2

- **Schema upgrade** (3,268 wallets, 55 active brands):
  - Per-wallet **evidence fields**: `evidence_type`, `evidence_source`, class-based `confidence_score`, `first_seen_at`, `as_of`
  - **Seed vs derived separation** with `is_seed_wallet` / `cluster_method`
  - **`volumeSuspectReasons`**: machine-readable why behind every under-review flag. Current: Rollbit + Gemini → `abnormal_avg_transfer_size`; Rain.gg → `manually_flagged_wash_or_treasury_pattern`; Gamba cleared from under-review this release
  - **`wallet_role` shipped** — behaviour-inferred over a rolling 14-day window: 174 `hot_wallet`, 1,489 `dormant`, 5 `deposit_address`, ambiguous → `null`. Refreshes continuously server-side.
- New file: `data/excluded-wallets.json` — the infra denylist, published so exclusions are auditable
- New docs: `DATA_DICTIONARY.md`, this changelog
- New: `scripts/export.mjs` — the full pipeline, reproducible by anyone from the public API
- Site API additions: `GET /api/labels/export` (per-wallet provenance + inferred roles), `volumeSuspectReasons` on `/api/brands`
